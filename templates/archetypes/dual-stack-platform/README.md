# Dual-Stack Platform Starter Kit

Best for:

- FastAPI + Next.js
- separate frontend/backend toolchains
- APIs with typed frontend contracts

Copy first:

- core repo OS files
- `TESTING_AND_BROWSER_AUTOMATION.md`
- schema-sync checklist
- incident runbook

Do not forget:

- separate backend/frontend commands in docs
- full-stack Playwright or smoke harness
- explicit model/schema/client sync rules
