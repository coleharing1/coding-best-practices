# Cursor Commands

Put shared Cursor commands in `.cursor/commands/`.

Use commands for stable prompts you want to invoke on purpose, such as:

- planning a feature
- synthesizing a final checklist
- reviewing the current diff
- verifying browser-visible behavior

Recommended naming pack:

- `plan-feature.md`
- `final-plan-checklist.md`
- `review.md`
- `browser-verify.md`

Keep command names aligned with `.claude/commands/` and `.gemini/commands/` when practical so the workflow stays portable.
