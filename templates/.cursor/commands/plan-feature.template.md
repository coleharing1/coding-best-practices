# Plan Feature

Write a plan for the requested feature as an owner-tagged checklist with blocking gates.

Requirements:

- use `plans/active/Plan-XXX-[Source].md`
- keep the plan concrete and phase-based
- include explicit human-only steps such as login, approval, or credential handoff
- call out risky files, schema changes, browser-visible flows, and rollback concerns

Checklist shape:

- `Context`
- `Current Evidence`
- `Working Assumptions`
- `Blocking Gates`
- `Implementation Checklist`
- `Review Plan`
- `Rollout / Backout`
- `Definition Of Done`
