# Debug Journal

A lightweight investigation log for bugs, outages, false leads, and root-cause work that should not all live in `WORKLOG.md`.

---

## Entry format rules

- Keep entries in reverse chronological order.
- Use real file paths, URLs, commands, and error text when useful.
- Record false leads only when they help future debugging.
