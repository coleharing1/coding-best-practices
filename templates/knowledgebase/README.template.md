# Knowledgebase

Use this folder for stable context that should outlive one feature plan.

Use `plans/MACRO-ROADMAP.md` for the durable whole-project build sequence and use this folder for the evergreen architecture, product, and policy context behind that roadmap.

Recommended starter files:

- `01-project-charter.md`
- `04-architecture-direction.md`
- `06-open-questions.md`
- `10-implementation-checklist.md`

Rule of thumb:

- put evergreen context here
- keep the whole-project roadmap in `plans/MACRO-ROADMAP.md`
- keep temporary implementation details in plans
- keep shipped change history in `WORKLOG.md`
